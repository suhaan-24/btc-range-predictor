# Bitcoin Next-Hour Range Predictor

**AlphaI × Polaris Build Challenge**

**Live App:** [btc-range-predictor.streamlit.app](https://btc-range-predictor.streamlit.app/)

Predicts the 95% confidence interval for BTCUSDT's next hourly close using a Geometric Brownian Motion (GBM) simulator with FIGARCH volatility modeling and Student-t fat-tailed innovations.

## Project Structure

```
├── model.py              # Core GBM model (data fetching, simulation, prediction)
├── backtest.py           # Part A: 30-day backtest over ~720 hourly bars
├── app.py                # Part B+C: Streamlit live dashboard with persistence
├── requirements.txt      # Python dependencies
├── backtest_results.jsonl # Backtest output (generated by backtest.py)
└── backtest_metrics.json  # Summary metrics (generated by backtest.py)
```

## Setup

```bash
pip install -r requirements.txt
```

## Part A — Backtest

```bash
python backtest.py
```

Runs predictions on ~720 hourly bars (last 30 days). Outputs:
- `backtest_results.jsonl` — one prediction per line
- `backtest_metrics.json` — coverage, avg width, Winkler score

## Part B — Live Dashboard

```bash
streamlit run app.py
```

Shows: current BTC price, predicted 95% range, price chart with ribbon, and backtest metrics.

## Part C — Persistence

Predictions are automatically saved to `prediction_history.jsonl` on each dashboard visit. Historical predictions show actuals once candles close.

## Key Concepts

1. **No peeking** — backtest loop strictly uses data up to bar N-1 to predict bar N
2. **Volatility clustering** — FIGARCH model captures calm/volatile regimes
3. **Fat tails** — Student-t distribution handles Bitcoin's extreme moves
